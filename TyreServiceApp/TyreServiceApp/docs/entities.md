# Сущности backend

## Ключевые бизнес-сущности

### `Client`

- хранит данные клиента
- связан с автомобилями
- используется в заказах и клиентском кабинете

Основные поля:

- `ClientId`
- `FullName`
- `Phone`
- `Email`

### `Car`

- принадлежит клиенту
- используется в заказах
- может содержать фото и дополнительные изображения

Основные поля:

- `CarId`
- `ClientId`
- `Brand`
- `Model`
- `ManufactureYear`
- `LicensePlate`
- `Vin`
- `PhotoPath`
- `AdditionalPhotos`

### `Order`

Центральная сущность бизнес-логики.

Основные поля:

- `OrderNumber`
- `OrderDate`
- `ScheduledAt`
- `PaymentDate`
- `Status`
- `CarId`
- `MasterId`
- `ClientTotal`
- `LaborCost`

Связи:

- один заказ относится к одному автомобилю
- один заказ может иметь много выполненных работ
- один заказ может иметь расходники
- один заказ может иметь коэффициенты сложности

### `CompletedWork`

Описывает конкретную выполненную услугу в заказе.

Основные поля:

- `WorkId`
- `OrderNumber`
- `ServiceCode`
- `MasterId`
- `WheelCount`
- `CompletionTimeMin`
- `WorkTotal`

### `Service`

Справочник услуг.

Основные поля:

- `ServiceCode`
- `ServiceName`
- `ServiceCost`

### `Master`

Основная модель исполнителя работ.

Поля зависят от текущей версии модели, но обычно включают:

- `MasterId`
- `FullName`
- `Position`
- `Rank`
- `HourlyRate`

### `Tire`

Хранит данные по шине автомобиля.

Основные поля:

- `TireId`
- `CarId`
- `TireType`
- `Seasonality`
- `Manufacturer`
- `TireModel`
- `Size`
- `LoadIndex`
- `WearPercentage`
- `Pressure`

### `CustomerReview`

Хранит клиентские отзывы, которые используются:

- в клиентском кабинете
- в административном интерфейсе
- в публичной части сайта

## Ролевые и пользовательские сущности

### `CustomerUser`

Учётная запись клиента для customer area.

### `MasterUser`

Учётная запись сотрудника для рабочего кабинета.

### `AdminUser`

Учётная запись административной части.

### `OwnerUser`

Учётная запись кабинета владельца.

### `StaffPosition`

Служебная сущность, связанная с должностями сотрудников административной части.

### `Position`

Справочник должностей.

## Операционные и справочные сущности

### `Post`

Справочник рабочих постов.

### `PostActiveSession`

Отражает активную сессию работы мастера на посту.

### `Consumable`

Справочник расходников.

### `CarClass`

Классификация автомобилей для тарифов и расчётов.

### `ServiceTariff`

Тариф услуги для конкретного класса автомобиля.

## Расчётные и связующие сущности

### `ComplexityCoefficient`

Коэффициенты сложности для расчёта стоимости заказа.

### `OrderConsumable`

Связь между заказом и расходниками с количеством и стоимостью использования.

### `OrderComplexity`

Связь между заказом и применёнными коэффициентами сложности.

### `WorkTimeLog`

Лог времени по выполненным работам.

### `CompletedJobsPayout`

Начисления или выплаты за выполненные работы.

### `SpeedBonus`

Сущность для бонусов, связанных со скоростью или эффективностью выполнения работ.

## Настройки и служебные сущности

### `OwnerSetting`

Настройки, используемые в кабинете владельца и смежных сценариях.
